# Writing Improvement Prompts

Use these prompts to ask an AI agent to improve the quality, clarity, and tone of written content.

---

## General Writing Polish

```
Please improve the following text. My goals are:
- [GOAL 1, e.g. "make it clearer and more concise"]
- [GOAL 2, e.g. "ensure a professional but approachable tone"]
- [GOAL 3, e.g. "fix any grammar or spelling issues"]

Preserve my original meaning and voice. Show me the revised version followed by a brief summary of the changes you made.

<text>
[PASTE TEXT HERE]
</text>
```

---

## Adjust Tone

```
Rewrite the following text with a [TARGET TONE] tone. (Examples of target tones: formal, casual, empathetic, assertive, persuasive, neutral, enthusiastic.)

Keep the core content and meaning intact. Show the rewritten version and briefly explain the main adjustments.

<text>
[PASTE TEXT HERE]
</text>
```

---

## Simplify for a Specific Audience

```
Rewrite the following text so it is easy to understand for [TARGET AUDIENCE, e.g. "a non-technical manager", "a 10-year-old", "someone new to the field"]. Avoid jargon, use simple vocabulary, and add analogies or examples where helpful.

<text>
[PASTE TEXT HERE]
</text>
```

---

## Shorten Without Losing Meaning

```
Please shorten the following text by approximately [TARGET REDUCTION, e.g. "30%"] without losing any important information. Cut filler phrases, redundant sentences, and unnecessary qualifiers. Do not omit key facts or change the meaning.

<text>
[PASTE TEXT HERE]
</text>
```

---

## Write a Summary

```
Please write a [LENGTH, e.g. "3-sentence" / "one-paragraph" / "bullet-point"] summary of the following content. Capture the most important points and the overall conclusion.

<content>
[PASTE CONTENT HERE]
</content>
```

---

## Technical Documentation

```
Write clear technical documentation for the following [code / API / feature]. Include:
- A one-line description
- A more detailed explanation of what it does and why
- Parameters / inputs and their types
- Return value / output
- Usage example(s)
- Any important caveats, limitations, or side effects

<subject>
[PASTE CODE / DESCRIPTION HERE]
</subject>
```

---

## Email / Message Draft

```
Draft a [professional / friendly / formal] [email / Slack message / comment] on my behalf.

**Context:** [DESCRIBE THE SITUATION]
**My main point:** [WHAT I NEED TO COMMUNICATE]
**Desired outcome:** [WHAT I WANT THE RECIPIENT TO DO OR UNDERSTAND]
**Tone:** [e.g. polite but direct]

Keep it concise — aim for [SHORT (2-3 sentences) or MEDIUM (1 paragraph)] length.
```
